# Ambitionz V1.06 Progression Loop Report\n\n- Version: Ambitionz V1.06 Progression Loop Pack\n\n## Core Loop\n\n### 1. Play Match\n\n- Purpose: Start the engagement loop with a quick duel.\n- Player feeling: I can play one more match quickly.\n\n### 2. Earn XP and Coins\n\n- Purpose: Reward time spent even after losses.\n- Player feeling: My time was not wasted.\n\n### 3. Complete Missions\n\n- Purpose: Give short-term goals beyond simply winning.\n- Player feeling: I have a reason to return today.\n\n### 4. Open Booster\n\n- Purpose: Create anticipation and collection growth.\n- Player feeling: Maybe I unlock something useful or rare.\n\n### 5. Improve Deck\n\n- Purpose: Turn rewards into strategic expression.\n- Player feeling: My deck is becoming mine.\n\n### 6. Climb Ranking\n\n- Purpose: Create competitive progression.\n- Player feeling: I am getting better and proving it.\n\n### 7. Unlock Cosmetic or Card Identity\n\n- Purpose: Create long-term attachment and personalization.\n- Player feeling: My account has history and identity.\n\n## Progression Systems\n\n### XP\n\n- Role: Measures account growth and unlock pacing.\n- Design rule: XP should reward both wins and participation.\n- Future use: Unlock levels, cosmetics, missions and beta milestones.\n\n### COINS\n\n- Role: Soft currency for boosters and future cosmetic purchases.\n- Design rule: Coins should feel useful but not inflate too fast.\n- Future use: Booster purchases, event entries, cosmetics.\n\n### MISSIONS\n\n- Role: Daily and onboarding goals.\n- Design rule: Missions must teach good gameplay behavior.\n- Future use: Daily retention, tutorial progression, beta tasks.\n\n### BOOSTERS\n\n- Role: Collection expansion and excitement.\n- Design rule: Boosters should support deck improvement without overwhelming new players.\n- Future use: Element packs, Sigil packs, event packs.\n\n### RANKING\n\n- Role: Competitive identity.\n- Design rule: Ranking should reward consistency, not only grind.\n- Future use: Seasons, tiers, leaderboard rewards.\n\n### COSMETICS\n\n- Role: Long-term personalization.\n- Design rule: Cosmetics should not affect gameplay power.\n- Future use: Card backs, frames, titles, avatars, arena skins.\n\n## Reward Philosophy\n\n- win: Winning should feel clearly better, especially in PvP.\n- loss: Losing should still advance the player slightly to reduce early churn.\n- draw: Draws should reward time but less than clean wins.\n- training: Training rewards should teach and encourage experimentation.\n- pvp: PvP rewards should be the main competitive progression source.\n\n## Mission Design Rules\n\n- A mission should teach or reinforce a useful behavior.\n- Avoid missions that force bad gameplay decisions.\n- Use simple verbs: play, win, use, complete, open, claim.\n- Early missions should be easy and fast.\n- Daily missions should create variety without forcing frustration.\n- Beta missions should generate testing data.\n\n## Booster Design Rules\n\n- Starter boosters should be readable and low complexity.\n- Element boosters should help players pursue identity.\n- Sigil boosters should help players pursue playstyle.\n- Rare cards should feel exciting but not mandatory.\n- Beta booster economy should be generous enough for testing.\n\n## Retention Targets\n\n- The first match should happen quickly.\n- The first reward should be immediate.\n- The first booster should be reachable early.\n- The first deck edit should feel meaningful.\n- The first ranking improvement should feel achievable.\n
